Structured workflow for collaboratively authoring documentation, specs, and test cases using AI.
Steps:
1. Analyze the current context, existing documentation, and recent code changes.
2. Identify missing, outdated, or inconsistent documentation.
3. Propose an action plan, including type of document needed (specs, README, tests, API docs).
4. Draft the document, incorporating technical details from the codebase.
5. Review and refine the draft based on feedback.
6. Validate the documentation against the codebase and requirements.
7. Commit changes with a clear message describing the documentation update.
